2

Penamaan (kunci, jangan ganti lagi)

Terkunci
  • Pre-order (tag ringkas "PO"). Buang "Pra-Pesan" dari build — terlalu formal, tidak konsisten.
  • Penjualan · Tagihan · Menu / Master Menu.
  • Status bayar: Belum Bayar (bukan "Belum Dibayar") & Lunas.
  • Aksi cetak: Print (bukan "Cetak"). Istilah natural Indo-Inggris, sesuai keputusan sebelumnya.
3

Menu (Master Menu) — Source of Truth

Terkunci
  • Tab utama, bukan submenu Pengaturan. Pusat seluruh data makanan, dibuat sekali.
  • Channel toggle: Pre-order ON/OFF & Penjualan ON/OFF — satu ketukan ikon. Pakai ikon Lucide (utensils / shopping-cart), bukan emoji 🍱🛒.
  • Varian + harga per varian (cth. S/M/L, Reguler/Jumbo). Menu tanpa varian = satu harga.
  • Kategori hanya sebagai filter pencarian.
  • Auto-save, tanpa tombol Simpan. Perubahan langsung berlaku di Pre-order & Penjualan.
Belum ada di build — ini layar paling penting yang masih kosong. Prioritas bangun nomor satu.
4

Pre-order — sisi Admin

Terkunci
  • Atas halaman: ringkasan hari ini + status Dibuka / Ditutup + Tanggal Layanan (satu tanggal aktif).
  • Bagikan Link (Salin · WhatsApp). Filter per kelas / per Waktu Ambil. Daftar pesanan.
  • Packing: status Sudah Dikemas / Belum Dikemas (buang "Sudah Diambil"). Ringkasan: jumlah pesanan · sudah · belum.
  • Rekap Masak (layar Mama): agregat on-screen "masak N porsi per menu", bisa difilter per Waktu Ambil — dan bisa di-Print.
  • Print di sini (operasional): Rekap per Kelas · per Menu · Label.
5

Pre-order — sisi Orang Tua

Terkunci
  • Link-only, hanya orang tua. Admin TIDAK mengisi atas nama orang tua.
  • Tanpa akun, tanpa dashboard, tanpa riwayat. Identitas: Nama Murid · Kelas · WhatsApp (tanpa ID Murid).
  • Layar akhir = Konfirmasi Pesanan (bukan "Receipt"), judul "Pesanan Berhasil Dikirim", isi blok Bukti Pesanan (No. Pesanan · Nama · Kelas · Tanggal Layanan · Waktu Ambil · Rincian · Total) + "simpan / screenshot sebagai bukti" → tombol Selesai.
  • Tiap submit = transaksi baru. Transaksi tidak dapat diubah; setiap transaksi menyimpan salinan nama + varian + harga saat dibuat (harga Master Menu berubah → transaksi lama tetap).
  • Pesanan via telepon/WA: kalau beli langsung → Penjualan; kalau untuk hari layanan → arahkan ke link Pre-order.
Kenapa: kalau admin boleh isi PO atas nama ortu, batas Pre-order vs Penjualan jadi kabur. Dipisah tegas: Pre-order = ortu via link, Penjualan = admin di kantin.
6

Penjualan — sisi Admin (live)

Terkunci
  • Hanya menu dengan channel Penjualan aktif. Alur: Pilih Menu → Ringkasan → Dibayar / Masuk Tagihan.
  • Popup Tagihan minta: Nama Siswa* · Kelas* · WhatsApp (opsional).
  • Auto catat Tanggal + Waktu transaksi. Tidak perlu pilih Tanggal Layanan (live).
7

Tagihan

+ Batalkan
  • Dua sumber → satu Tagihan: Pre-order yang belum dibayar + Penjualan "Masuk Tagihan". Semua transaksi belum bayar, dikelompokkan per murid. Rincian tampil langsung.
  • Cari: Nama · Kelas · WhatsApp (untuk kasus kakak-adik). Tandai Lunas (dengan Undo). Bagikan WhatsApp.
  • Batalkan Transaksi — bukan edit. Karena transaksi tidak dapat diubah, ini satu-satunya cara koreksi salah pesan. Riwayat pembatalan tetap disimpan sebagai implementasi di belakang layar.
8

Pengaturan (gear di header)

Terkunci
  • Hanya config jarang berubah: Nama Kantin · Printer · Nomor WhatsApp · Backup · Tentang.
  • Status Pre-order & Tanggal Layanan TIDAK di sini — itu operasional harian, tempatnya di halaman Pre-order. (Build sekarang salah menaruhnya di Settings — perlu dipindah.)
9

Tanggal Layanan & Waktu Ambil

Terkunci
  • Satu Tanggal Layanan aktif pada satu waktu. Admin ganti via date picker; seluruh halaman Pre-order & Parent otomatis ikut. Parent tidak memilih tanggal.
  • Waktu Ambil dipilih orang tua saat Pre-order, dari preset (cth. Sebelum masuk · Istirahat 1 · Istirahat 2 · Pulang Sekolah). Tanpa jam spesifik.
  • Preset Waktu Ambil dikonfigurasi di halaman PO (operasional harian), bukan di Pengaturan.
  • Satu transaksi = satu Waktu Ambil. Beda waktu → transaksi terpisah.
  • Rekap & Packing bisa difilter per Waktu Ambil — supaya dapur tahu "masak berapa untuk Istirahat 1".

A

Penyelarasan Build dengan Decision Lock

Teknis
  • Tukar theme.css ke token palet final + sapu kelas hardcode (amber-500, text-white di tab aktif, emerald/red-500).
  • Teks/ikon di amber → gelap (build sekarang putih — masalah kontras 50–60).
  • Buang 🔔 Bell / notifikasi — scope creep, tak ada di PRD.
  • Emoji 🍱/🛒 → ikon Lucide.
  • Pindahkan Status Pre-order & Tanggal Layanan keluar dari Settings → ke halaman Pre-order.
  • Tambah tab Menu; ganti label "Pra-Pesan" → "Pre-order".
  • Bangun yang masih kosong: Master Menu, varian, channel, Print/Rekap/Label, packing, sisi Parent + Receipt.
B

Catatan Teknis (untuk fase implementasi)

Teknis
  • Print ke printer thermal (Bluetooth/USB) dari web tidak trivial — memengaruhi pilihan PWA vs native Android. Bukan penghalang sekarang; diputuskan saat mulai fase implementasi print.
